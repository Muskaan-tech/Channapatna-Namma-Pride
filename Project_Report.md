# Project Report: Channapatna Namma Pride

## 1. Aim
To develop an e-commerce platform dedicated to Channapatna wooden toys that empowers local artisans by connecting them directly with consumers. The system aims to eliminate counterfeit products through a unique 6-digit verification system, ensuring authenticity, preserving cultural heritage, and enhancing the shopping experience with modern technologies like Augmented Reality (AR) and storytelling.

## 2. Problem Statement
The traditional Channapatna toy industry, famous for its eco-friendly wooden crafts and natural dyes, is facing a severe threat from cheap, mass-produced counterfeit products in the market.
* **Current challenges faced by users:** Consumers struggle to differentiate between genuine handcrafted toys and fake alternatives. Artisans lose out on fair compensation and recognition due to middlemen and counterfeiters.
* **Limitations of existing solutions:** Most e-commerce platforms do not have built-in verification systems for handcrafted goods. They also fail to tell the story behind the product or adequately showcase the artisan's profile.
* **Why this problem is important:** Preserving this centuries-old craft is crucial for the livelihood of artisans in Karnataka. Solving this problem restores consumer trust, provides fair wages to creators, and sustains cultural heritage.

## 3. Objectives
* **To develop** a user-friendly e-commerce web and mobile application for browsing and purchasing Channapatna toys.
* **To implement** a secure 6-digit authentication system allowing users to verify the genuineness of their purchased products.
* **To integrate** Augmented Reality (AR) functionality, enabling users to preview 3D models of toys in their physical space before buying.
* **To design** artisan profile pages that highlight the creator's biography, experience, and portfolio, fostering a direct connection with the buyer.
* **To create** a storytelling feature that generates engaging narratives about the crafting process and heritage for each product.

## 4. Methodology
The project follows a component-based architecture for a seamless user experience, functioning as a Progressive Web App (PWA).

**Technologies Used:**
* **Frontend:** React.js, Vite, HTML5, Vanilla CSS (for custom, responsive styling).
* **Mobile Wrapper:** Capacitor.js (to convert the web app into a mobile application).
* **AR Viewer:** `<model-viewer>` web component by Google for displaying 3D `.glb` files with WebXR support.
* **Data & State Management:** React Hooks (`useState`, `useEffect`) and modular simulated backend (`data.js`).

**Step-by-Step Implementation Flow:**
* **Input from user:** Users interact via touch/click inputs for navigation, text input for search and the 6-digit verification code, and camera permissions for the AR viewer.
* **Processing (Logic/Model):**
  * *Verification Logic:* The 6-digit code entered by the user is cross-referenced against a registered database array (`verificationCodes`). If a match is found, it maps to the exact `productId` and `artisanId`.
  * *AR Preview:* The `model-viewer` component loads local `.glb` 3D assets mapped to specific product IDs. It processes device sensor data for AR placement via Scene Viewer (Android) or AR Quick Look (iOS).
  * *Story Generation:* Connects product and artisan metadata to present a personalized, engaging narrative about the toy's creation.
  * *Shopping Cart:* React state manages the addition of products to the cart and calculates pricing dynamically.
* **Output/response:** Real-time UI feedback for authentication (Success/Error modal), rendering of 3D models in the physical environment, dynamic rendering of cart totals, and pop-up toast notifications ("Added to cart!").
* **Data storage:** Currently, the system utilizes simulated local JSON-like data arrays (`products`, `artisans`, `verificationCodes`) structured similarly to a NoSQL database (e.g., Firebase Firestore) for rapid MVP deployment, testing without network latency, and offline capability.

## 5. Results
The implemented system successfully bridges the gap between traditional Channapatna artisans and modern digital consumers.

* **Output generated by system:** A highly responsive e-commerce application with functioning product discovery, AR previews, cart management, and product authentication.
* **How it solves the problem effectively:** The 6-digit verification completely mitigates the risk of counterfeit products by acting as a digital certificate of authenticity. The artisan profiles humanize the purchase, cutting out the anonymity caused by middlemen.
* **Benefits to users:** Buyers receive guaranteed authentic products and an interactive shopping experience. Artisans receive digital visibility and direct attribution for their hard work.
* **Performance insights:** By utilizing Vite and local asset loading for GLB files and high-quality portraits, the application achieves near-instantaneous load times. The PWA architecture allows it to function smoothly on both desktop and mobile devices.

**Real-world Example Scenario:**
* **User input:** A user buys a "Wooden Rocking Horse" physically from a store and finds a tag with the code "731456". They open the app, navigate to the "Verify Authenticity" screen, and enter the code into the OTP boxes.
* **System behavior:** The app queries the database, finds an active match for "731456", and retrieves the linked product ("Wooden Rocking Horse") and artisan ("Ravi Shankar").
* **Final outcome:** A green "Verified!" success card slides up on the screen, showing the exact product and artisan name. This assures the user that the item is a genuine Channapatna craft and allows them to click through to read the artisan's personal story.
